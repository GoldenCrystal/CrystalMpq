CrystalMpq

By GoldenCrystal, Updated for WoW 3.1.1

What is it ?
CrystalMpq is a C# library designed to open MPQ archives. It still doesn't support all compression types, but most files should open correctly. WoW Archives don't use the older compression types and should never encounter any problems. Older game archives such as Warcraft III archives might use unsupported compressions (such as ADPCM or Huffman that aren't supported yet), meaning that some files might be unreadable by CrystalMpq for the time being.
Some applications were developped along with the library and are part of the package. They should allow you to peek into your favorite Blizzard MMORPG's archives. Older game's archives should work fine as long as you don't want to access WAVE data.

Requirements
  • Windows 98 or later
  • Microsoft .NET Framework 2.0
  • DirectX 9.0c (for Audio/Video visualization)
  • Managed DirectX Runtime (for Audio/Video visualization)
How to install
Unpack the whole archive to a directory of your choice, and ensure that you meet the above requirements.
You might find it useful to run the dxwebsetup.exe tool. It will update your DirectX installation by downloading needed files directly from Microsoft's site.
Applications included in the package
  • CrystalMpqExplorer: An explorer-style interface designed to browse MPQ files and their content.
  • DbcViewer: A simple user interface application that lets you open separate WoW DBC files.
  • WoWMapExplorer: A simple World Map viewer mimicking the World Map feature of WoW.
  • WoWSpellViewer: A basic interface for browing WoW Spells. Still Work in progress.
Known issues
  • There is a bug (leading to a crash) if you ever try to open BLP textures smaller than 4x4 pixels. It's just a stupid buffer overflow bug but i'm too lazy to fix it for now. I'll have to rewrite most of the texture loading code at some point anyway, so it'll probably be done at the same time. Because of this bug, loading of mipmaps in BLP Textures is disabled.
  • The DBC viewer can be quite memory hungry because of the usage of .NET 2.0's DataGridView control. Please note that trying to open files too large may easily lead your computer to run out of free memory and degrade considerably your user experience.
    This remarks apply particularly to the file "Spell.dbc" that has now grown over 40 MB and cannot be viewed anymore, sorry. For your convenience, a warning has been added to the DBC viewer to prevent you from viewing too large files. You are free to bypass this warning, but be aware that it might be a really bad idea for some files, and I advise you never to bypass it for the "Spell.dbc" file.
    Anyway, I canot be held responsible for any consequences of the bypass of this warning...
  • Missing (known) compression types: ADPCM, Huffman
  • It is not possible to manually pick a WoW installation directory for now...
  • Some parts of the code are really old, and may need some refactoring or event complete rewriting, and thus, some "small" features might need more work than expected to implement.
